
# 🕷 Краулер Telegram-бота

Умный скрипт для обхода всех меню Telegram-бота. 
В отличие от встроенного парсера, этот **не заходит дважды** в одни и те же вкладки.

## ⚙️ Настройка

### 1. Получите API_ID и API_HASH
1. Зайдите на [my.telegram.org](https://my.telegram.org)
2. Войдите по номеру телефона
3. Перейдите в **API development tools**
4. Создайте приложение (любое название)
5. Скопируйте `api_id` и `api_hash`

### 2. Впишите их в `crawler.py`
Откройте `crawler.py` и заполните строки в начале:
```python
API_ID = 1234567        # ваш api_id (число)
API_HASH = "abc123..."  # ваш api_hash (строка)
```

### 3. Укажите бота
В `crawler.py` уже указан нужный бот:
```python
BOT_USERNAME = "zipppppppppaaaabot"
```

### 4. Установите зависимость
```bash
pip install telethon==1.36.0
```

### 5. Запустите
```bash
python crawler.py
```

При первом запуске потребуется авторизация:
- Введите номер телефона
- Введите код из Telegram/SMS
- (если включена) введите пароль 2FA

## 🧠 Как работает логика

```
1. Отправляет /start → собирает текст и кнопки
2. Добавляет все кнопки в очередь (BFS)
3. Нажимает каждую кнопку ОДИН РАЗ (visited set)
4. У новых сообщений снова собирает кнопки
5. Повторяет до глубины MAX_DEPTH (3)
6. Сохраняет результат в JSON и TXT
```

### Ключевые отличия от встроенного:
| Проблема | Решение |
|----------|---------|
| Заходит по 10 раз в одни вкладки | `visited` set — каждая кнопка нажимается **1 раз** |
| Нет ограничения глубины | `MAX_DEPTH = 3` — чёткий лимит |
| Хаотичный обход | BFS (обход в ширину) — по уровням |
| Флуд-вейты | `DELAY = 1.5 сек` между нажатиями |
| URL-кнопки открывают ссылки | URL-кнопки пропускаются |

## 📁 Результат

После обхода создаются 2 файла:
- `crawl_result_YYYYMMDD_HHMMSS.json` — полная структура (для программы)
- `crawl_result_YYYYMMDD_HHMMSS.txt` — читаемое дерево (для вас)

## 🔄 Что делать после обхода

1. Откройте `crawl_result_*.txt` — посмотрите структуру
2. Пришлите мне содержимое JSON-файла
3. Я перепишу `main.py` под **точную** структуру оригинального бота
